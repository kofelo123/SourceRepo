###### 180817_7

운영체제 개요
-

(INDEX SCAN)

컴퓨터의 사용자와 HW 사이의 중재자 역할을 한다.


수작업 -> 단일 프로그래밍 -> 다중 프로그래밍 &시분할 -> 모바일 OS & 실시간 시스템

운영체제의 목적

1.사용자가 컴퓨터를 편리하게 사용

2. 컴퓨터 하드웨어가 효율적으로 사용되게 함



하드웨어 위에 OS 위에 APPLICATION 위에 사용자가 있다.

좋은 운영체제는 자원을 효율적으로 할당해주는 자원 할당자 역할을 하는 운영체제.

공정하고 효율적이게 운영할 수 있도록 적절한 자원을 할당하고 입/출력을 제어한다.

###### 180817_8

시스템의 발전
-

일괄처리 시스템 -> 다중 프로그램 & 시분하 시스템으로의 발전


<일괄 처리 시스템(Batch System)

:초기의 컴퓨터가 동작하는 방법, 즉시 데이터를 처리할 수있는 여건을 갖추지 못했다. 그 대안으로
일괄처리 시스템이 os에 도입됨으로써 비슷한것은 주기적으로 한 번에 묶어서 처리하도록 했다.

유사한 요구를 가지는 작업을 모아서 하나의 그룹으로 수행한다.

입/출력 장치의 속도가 cpu와 같은 전자적 장치의 속도보다 느려서 cpu가 계속 쉬는상태인 경우가 많았다.

상호작용이 필요없고 한꺼번에 처리하는 시스템이 필요한 경우 요즘에도 이런기술을 쓰는 운영체제가 있긴하다.


<다중 프로그램 시스템 (Multi-Programmed System)>

: CPU가 수행할 작업을 항상 가지도록 하는 방식.

여러개의 프로그램을 선택해서 메인 메모리(RAM)에 적재한다.(실행중인 프로그램)

수행중인 작업이 입/출력등  떄문에 기다리는 상태가 되면 cpu는 다른작업으로 넘어가서 수행을 게속한다.

cpu는 계속일을 하기때문에 자원의 효율적인 배분이다.

<시분할 시스템(Time-Sharing System)>

: 아주 짧은 주기로 cpu를 각각의 프로그램에 할당을 해준다 -> 프로그램이 수행되고 있는 동안에도 상호작용을 할 수 있다. 

아주 짧은 주기로 전환이 이루어져 각 사용자는 여러 프로그램이 동시에 작동을 하고 있다고 느끼게 된다.

여러사람이 한대의 컴퓨터에 접속해서 각기 다른일을 해도 개인은 본인이 독점해서 사용하는것 같은 느낌을 준다.


<작업 스케줄링> 

보조기억장치(하드,ssd 등)되어 있고 그 중 일부를 선택해서 메모리에 적재하는 전략

<cpu 스케줄링>

메모리에 올라온 작업들 중에서 무엇부터 실행할 것인지 고르는 전략

###### 180821_1

다양한 시스템
-

<다중 처리 시스템(Multiprocessor System)>
: cpu가 여러 개인 시스템을 의미하며 각각의 cpu들이 아주 밀접하게 통신하는 구조

cpu을 여러개 가지므로 성능이 더 좋고

하나가 고장나도 속도가 느려질뿐 시스템은 정상으로 작동가능

흔히 두가지로 분류

- 대칭적 다중 처리 시스템: 각 cpu가 하나의 운영체제 하에서 작동 cpu 끼리 데이터를 효율적으로 공유하는 구조로 운용됨.


- 비대칭적 다중 처리 시스템: 각 처리기마다 특정 작업이 정해져 있고 하나의 주요 cpu가 시스템을 제어하며 다른 cpu들은 미리 정의된 작업 혹은 명령을 수행
(Master-Salve구조)

<분산 처리 시스템(Distributed Processing System)>

네트워크를 이용해 동시에 작업을 처리한다.

각 cpu들은 메모리를 공유하지 않는다. 

적절한 자원공유로 계산 속도가 증가하고 신뢰성이 증가한다.(하나가 망가져도 운용에 문제없음)

그 중

- 클라이언트 & 서버 시스템: 클라-서버 관계로 되는 구조. 대표적 예로 웹 사이트와 접속자 사이의 관계

- p2p 시스템: 각 컴퓨터가 서버이자 클라이언트가 되는 시스템, 토렌트서비스가 있다.
(피어를 가지는 각 사람들이 서버가 되어 보내주는식)


<클러스터 시스템(Cluster System)>: 물리적으로 많은 CPU를 한 곳에 모아서 특정한 작업을 수행하는 시스템 . LAN(물리적으로 한곳에 모음)으로 연결된 각 cpu가 서로 상호작용하여 고속처리 서비스.

그 중

- 대칭적 클러스터 시스템 : 각 CPU가 호스트가 되어 서로를 관리하여 둘 이상작업을 수행

- 비 대칭적 클러스터 시스템: 호스트cpu가 다른 cpu를 관리하는 마스터 슬레이브 관계.


<실시간 시스템(Real-TIme System)>: cpu의 동작이나 작업이 즉시적인 처리를 요할때 채택되는 시스템. 의학영상시스템,무기시스템등


그중

- 연성 실시간 시스템(Soft Real-Time System) : 우선순위를 가지고 빠르게 계산되긴 해야하지만 절대적으로 시간만을 중요시하지는 않는 시스템 ->음악 등을 들을때 렉이 걸려도 끊기지 않음


- 경성 실시간 시스템(Hard Real - Time System): 빠른 시간이 절대적으로 요구됨. 단기 메모리만 써서 시간엄수. 미사일 발사 시스템 등 정밀하고 신속한 판별 요구할때 사용.

###### 180821_2

컴퓨터의 작동 과정
-

cpu와 각 장치들은 서로 연결되어 메모리를 공유한다.

장치 제어기가 작업을 도와준다.


<컴퓨터 부팅과정>

전원 켜질때 ROM에 저장된 초기 프로그램 실행
-> 메모리, cpu 레지스터 등을 초기화

운영체제 찾아서 메모리에 적재
-> 첫번째 프로세스를 즉시 실행

인터럽트가 발생하여 cpu가 각종 작업을 처리한다.

인터럽트: 이벤트를 말한다. cpu가 다음작업을 해야할것을 알려주는.. (타이핑을 하는것도 인터럽트)

인터럽트 발생시 cpu는 하던일을 중단하고 처리를 한다. '서비스 루틴'이라고 한다.

일반적으로 os가 이를 처리한다.

###### 180821_3

입출력 시스템
-

<입력 시스템의 작동과정>

cpu가 장치제어기의 레지스터에 명령을 적재

장치제어기는 레지스터 내용을 조사하여 수행할 동작을 검사

레지스터 내용이 읽기 명령이면 입력 장치에서 읽기 신호를 보낸다

입력 장치는 데이터를 읽어서 버퍼에 저장

제어 부분은 cpu에 인터럽트를 발생

cpu가 버퍼에서 데이터를 읽어 들이고 입력이 완료됨.


장치제어기: 공통 버스와 장치 사이에 위치하여 장치를 제어하는 역할. 내부적으로 저장장치를 가지고 있을 수 있고, 주변장치와 데이터 송수신을 하도록 인터페이스역할


동기식 입출력: 입출력이 완전히 끝난후에 제어가 사용자 프로세스로 복귀.

여러장치에 대한 동시 입출력이 불가능하며, 병행 실행될 가능성을 완전히 배제


비동기식 입출력: 입출력이 시작되고 끝나기전에 계속 다른 일을 수행할 수있다. 효율성 증가

운영체제가 대기큐를 유지해서 다른작업들을 병행하도록 한다.



<DMA(Direct Memory access)구조>

데이터를 낮개가 아니라 블록단위로 효율적으로 전송

cpu의 간섭이 없고 입출력작업을 처리한다.

고속 입출력 장치를 위해 사용, 기존에 발생한 cpu와 입출력장치의 속도차로 인한 병목현상을 개선하는 기술로 활용.

###### 180821_4

저장장치 구조
-

프로세스 : 실행 중인프로그램

주기억장치 : 프로세스가 올라가는 공간 주기억 장치와 RAM은 같은 의미로 사용된다.

cpu와 상호작용을 하며

<하나의 명령이 실행되는 사이클>

1. 기억장치로부터 명령을 인출해서 cpu 내부의 명령 레지스터에 저장

2. cpu가 명령을 해독

3. 피연산자를 cpu 내부의 레지스터에 저장

4.명령의 실행

5.결과를 기억장치에 저장


보조기억장치보다 용량이 적은대신 연산이 빠르다. 전원 공급되지 않으면 내용을 잃어버리는 휘발성 저장장치

주기억장치와 cpu자체에 내장되어 있는 레지스터는 cpu가 직접 접근할 수있는 유일한 저장장치이다.

입출력장치도 데이터 보관을 위해 내부적으로 레지스터를 가지고cpu가 명령을 내려 서로 상호작용도 한다.
